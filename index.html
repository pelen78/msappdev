Project

Build & Launch a Web App from Scratch
(AI + GitHub + Cloudflare)

In this project, you will build a web app from zero and publish it publicly—like a real product launch. You will use an AI design/editor tool (Antigravity or similar) to create branding and UI ideas, then code the app, document it on GitHub, and deploy it with Cloudflare Pages.

What You'll Learn

Learning Goals

By the end of this project, you'll have real skills that actual developers use every day.

  • 01
    Design a basic brand identity using an AI tool — name, logo, colors, and UI ideas
  • 02
    Build a functional web app using HTML, CSS, and JavaScript
  • 03
    Use GitHub to manage a repository and publish your code
  • 04
    Deploy a live site using Cloudflare Pages
  • 05
    Document your app like a real developer — README, screenshots, changelog
  • ★
    Optional: Connect a custom domain and configure DNS settings
What's Required

Minimum Requirements

Your app must meet all 7 of these requirements to be considered complete.

  1. A clear purpose — solve a small problem or provide a useful tool
  2. A homepage with a title, 1–2 sentence description, and navigation/buttons
  3. One main feature that fully works
  4. A clean UI — consistent colors, readable text, good spacing and alignment
  5. Responsive layout that looks okay on a phone-sized screen
  6. At least 2 empty/error states (e.g., "No results yet", "Please enter a value", "Something went wrong")
  7. Public deployment using Cloudflare Pages
Option A

API-Based App

Pull live data from a public API — weather, quotes, dictionary, currency converter, NASA imagery, and more. No backend required.

Option B

Local App

Build a fully self-contained app — habit tracker, planner, timer, checklist, or inventory manager. Use LocalStorage to persist data.

💡
Keep scope realistic. One strong feature is better than five broken features. Focus on doing one thing well before adding extras.
Step by Step

Project Phases & Deliverables

Follow these phases in order. Each one builds on the last.

1
Phase 1

App Concept + Plan

1–2 classes
  • Write a 1-page App Brief including: app name, problem it solves, target user, main feature, up to 2 bonus features, and success criteria
Checkpoint: Teacher approval before moving on
2
Phase 2

Branding + UI Design Using AI

~1 week
  • App name + tagline
  • Logo in 2 versions (light and dark or full/icon)
  • Color palette with 5 hex codes
  • Font selection (1–2 fonts)
  • 2 UI mockups — Home screen and Main Feature screen
  • Short copy: About section and button/label text
Checkpoint: Consistent look and readable contrast
3
Phase 3

Build the MVP

2–3 weeks
  • Required files: index.html, styles.css, app.js, assets/
  • Homepage and navigation are complete
  • Main feature is fully working
  • Layout is responsive on mobile
  • At least 2 empty/error states are handled
  • 1 polish improvement: loading state, validation, or card UI
Checkpoint: Teacher tests the app locally
4
Phase 4

GitHub Repo + Documentation

~1 week
  • GitHub repository with multiple commits (showing your work over time)
  • README includes: purpose, features, 2–4 screenshots, live link, and "run locally" steps
  • Changelog with versions: v0.1, v0.2, v1.0
Checkpoint: Repo is readable and complete
5
Phase 5

Deploy on Cloudflare Pages

~1 week
  • Cloudflare Pages connected to your GitHub repo
  • Live public URL generated and working
  • QA checklist fully completed
  • Final polish pass done
Checkpoint: Teacher verifies the live link
★
Phase 6 — Optional

Custom Domain + DNS

  • Option A (recommended): Use the Cloudflare Pages URL as your final link — no extra setup needed
  • Option B: Connect a custom domain — domain connected, DNS screenshot as evidence, final domain URL submitted

Using AI Responsibly

AI Tool Guidelines

AI is a tool, not a shortcut. Use it to level up your work—not replace your thinking.

What AI Can Help With

  • Branding ideas — name, tagline, mood
  • UI concepts and layout directions
  • Logos and icons
  • Short copy — About section, labels, button text
  • Code suggestions — as long as you understand what it does

What You Must Do

  • Mention AI use in your README — be specific
  • Be able to explain any code in your project
  • Never submit AI-generated code you don't understand
