GitHub Setup

Get your code
on GitHub

6 simple steps — from creating your account to uploading your first files. No command line needed.

1
Step 1

Create your GitHub account

Go to github.com and click Sign up. Use your school email and pick a username that looks professional — you'll be sharing this with others.

💡 Your username will appear in all your project links. Keep it simple: firstname-lastname or just your first name works great.
2
Step 2

Create a new repository

Once you're logged in, click the + icon in the top-right corner of the page, then select New repository.

A repository is like a folder for your entire project — all your files live here.

3
Step 3

Name your repository

Pick a short, clear name that describes your app. Rules: lowercase only, no spaces, use hyphens instead.

✓ Good names weather-app
todo-list
my-portfolio
✗ Avoid My App Final v2
project123
asdfgh
4
Step 4

Configure the repo

Before clicking Create, make sure to:

✅ Set visibility to Public — required for Cloudflare Pages deployment.
✅ Check Add a README file — this initializes your repo so you can upload files right away.

Then click the green Create repository button.

5
Step 5

Upload your files

Inside your new repo, click Add file → Upload files. Drag and drop your project files (index.html, styles.css, app.js, and your assets/ folder).

⚠️ Make sure index.html is at the root level (not inside a subfolder), otherwise Cloudflare Pages won't find it.
6
Step 6

Commit your changes

Scroll down to the Commit changes section. Write a short message that describes what you uploaded — for example: "Add initial project files".

Then click Commit changes. Your files are now live on GitHub! 🎉

💡 Every time you update your project, upload the new files and write a clear commit message. This gives you a history of your progress.
